# CafeFlow ☕
CafeFlow is a full-stack restaurant ordering and management web application built to simulate how modern food ordering platforms work.  
The platform allows users to explore restaurants, browse menus, add items to a cart, place orders, and track their order history through a clean and interactive interface.

This project demonstrates how a complete web application works by combining frontend development, backend API creation, and database integration.

---

# Project Purpose

The main goal of CafeFlow is to understand and implement a full-stack application architecture.

Through this project, I aimed to learn how different technologies interact together in a real-world application. The project focuses on building a system where users can browse restaurant menus and place orders while the backend handles authentication, data storage, and order processing.

Key learning objectives included:

• Building dynamic and responsive user interfaces  
• Implementing authentication systems  
• Designing RESTful APIs  
• Connecting frontend and backend services  
• Managing application state and user sessions  
• Working with databases for storing user and order data  

This project helped strengthen my understanding of full-stack development using modern web technologies.

---

# Key Features

### User Authentication
Users can create an account and log in securely to access the platform.

### Restaurant Browsing
Users can view a list of available restaurants and explore their menus.

### Menu Viewing
Each restaurant has a dedicated menu page displaying available food items.

### Cart Management
Users can add multiple items to a cart before proceeding to checkout.

### Checkout Process
The system allows users to confirm their orders through a checkout page.

### Order Confirmation
After placing an order, the user receives confirmation of the order placement.

### Order History
Users can view their past orders and track previously placed orders.

### Admin Dashboard
Administrators can monitor and manage system operations through an admin interface.

# Tech Stack

## Frontend
The frontend of the application is responsible for the user interface and user experience.

Technologies used:

• React  
• Vite  
• CSS  

React enables dynamic UI rendering while Vite provides a fast development environment.

---

## Backend
The backend handles server-side logic, API endpoints, and communication with the database.

Technologies used:

• Node.js  
• Express.js  

Express is used to build REST APIs that allow the frontend to interact with the server.

---

## Database
The database stores user data, menu information, and order records.

Technology used:

• MongoDB

MongoDB allows flexible and scalable storage of application data.

---

# Project Architecture

CafeFlow follows a **client-server architecture**.

Frontend handles:

• User interface  
• Page routing  
• Cart state management  
• API requests  

Backend handles:

• Authentication  
• Order management  
• Database operations  
• API endpoints  

Database stores:

• User accounts  
• Menu data  
• Order history
